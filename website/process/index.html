How It Works

Every application goes through automated and manual review. We verify everything.

The Process

1

Application Detected

Our scanner monitors the treasury 24/7. When you send an application transaction, we detect it within minutes. Your wallet address, Twitter handle, GitHub, and application data are automatically extracted.

2

Identity Verification

We verify you're real and legitimate:

  • Twitter account age and activity patterns
  • GitHub commit history and code quality
  • Wallet transaction history
  • Cross-reference all data points
3

Product Analysis

We evaluate your agent:

  • Is the product live and functional?
  • Code quality review (if open source)
  • User activity and traction metrics
  • Revenue evidence if claimed
4

Risk Assessment

We check for red flags:

  • Duplicate application detection
  • Known scam wallet screening
  • Social graph analysis
  • Red flag pattern matching
5

Economic Review

We assess the business:

  • Revenue model viability
  • Market size and competition
  • Sustainability assessment
  • ROI potential
6

Decision

Within 24 hours, we make a decision:

  • Approved: Funds sent to your wallet + Twitter announcement
  • Needs Info: We reach out with questions
  • Rejected: No response (we don't explain rejections)

What We Look For

Must Have

  • Working product (not an idea or prototype)
  • Clear path to revenue
  • Verifiable identity
  • Active development

Red Flags

  • New Twitter/GitHub accounts with no history
  • Copied or forked code with no original work
  • Inflated or fake metrics
  • Vague revenue model
  • Previous scam associations

Bonus Points

  • Existing revenue or users
  • Strong technical implementation
  • Unique value proposition
  • Active community engagement
